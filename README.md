# Gamified Transaction Hub - Internship Selection Task

This repository contains the full-stack solution for the Internship Selection Task, built with a focus on production readiness, strong backend fundamentals, and a unique, premium gamified frontend.

## 🚀 Features & Technical Decisions

### Backend (Python / FastAPI)
- **Framework**: FastAPI for high performance, automatic OpenAPI documentation, and asynchronous capabilities.
- **Database**: PostgreSQL (using asyncpg & SQLAlchemy 2.0) for robust data consistency and concurrent transaction handling.
- **Concurrency & Fairness**: Row-level locking (`SELECT ... FOR UPDATE`) is used when updating a user's total amount and score to prevent race conditions during simultaneous requests.
- **Idempotency**: Prevent duplicate processing by requiring an `idempotency_key` (UUID generated by the frontend). Database `UNIQUE` constraints ensure that even if multiple requests bypass application logic, the database will safely reject duplicates.
- **Abuse Prevention**: Rate limiting is implemented using `slowapi`. Furthermore, the ranking system only awards the flat transaction bonus points if the transaction amount is >= $1.00, preventing users from spamming micro-transactions ($0.01) to unfairly inflate their score.
- **Ranking Logic**: `Score = Total Amount + (Valid Transactions * 10)`.

### Frontend (React + Vite)
- **Architecture**: A clean, multi-page layout managed by `react-router-dom` separating concerns into Dashboard, Transactions, and Activity views.
- **UI/UX & Premium Aesthetics**: A highly polished, custom "Glassmorphism" UI enhanced with Framer Motion. It features a liquid navigation bar, smooth page transitions, staggered cascading data lists, dynamic animated background orbs, and deep hover interactions to provide a truly premium experience.
- **Idempotency Integration**: The frontend automatically generates and passes an `idempotency_key` via the `uuid` package for every transaction.

## 🛠️ How to Run Locally

### Prerequisites
- Node.js (v18+)
- Python (3.10+)
- PostgreSQL Server

### 1. Backend Setup

1. Open a terminal and navigate to the `backend` folder:
   ```bash
   cd backend
   ```
2. Create and activate a virtual environment:
   ```bash
   python -m venv venv
   # On Windows
   .\venv\Scripts\activate
   # On Mac/Linux
   source venv/bin/activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Setup PostgreSQL Database:
   - Create a PostgreSQL database named `internship_db`.
   - Copy `.env.example` to `.env` and update `DATABASE_URL` with your credentials:
     ```
     DATABASE_URL=postgresql+asyncpg://<username>:<password>@localhost:5432/internship_db
     ```
5. Run the backend server:
   ```bash
   uvicorn app.main:app --reload
   ```
   *(The server will start on `http://localhost:8000`. Swagger docs available at `http://localhost:8000/docs`)*

### 2. Frontend Setup

1. Open a new terminal and navigate to the `frontend` folder:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Create a `.env` file in the `frontend` directory (optional, defaults to localhost:8000):
   ```
   VITE_API_URL=http://localhost:8000
   ```
4. Start the Vite development server:
   ```bash
   npm run dev
   ```
   *(The app will start on `http://localhost:5173`)*

## 📖 API Documentation

### `POST /transaction`
Processes a new transaction.
- **Body**:
  ```json
  {
    "user_id": "string",
    "amount": 15.5,
    "idempotency_key": "uuid-string"
  }
  ```
- **Concurrency**: Locks the user row in PostgreSQL during update.
- **Idempotency**: Returns `400 Bad Request` (or the existing success response) if the `idempotency_key` has already been used.

### `GET /summary/:userId`
Retrieves a user's total amount, transaction count, and score.
- **Response**:
  ```json
  {
    "id": "string",
    "username": "string",
    "total_amount": 100.0,
    "transaction_count": 2,
    "score": 120.0
  }
  ```

### `GET /ranking`
Retrieves the top users ordered by score.
- **Query Params**: `limit` (default: 10)
- **Response**: Array of users with their rank and score.
